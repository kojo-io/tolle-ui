A migration runs in three phases. First the schema is copied to a shadow table, which is cheap because nothing is written yet. Then rows are backfilled in batches, with the batch size tuned down whenever replication lag crosses a threshold. Finally the tables are swapped inside a transaction, which is the only moment writes are blocked, and it lasts a few milliseconds. If the swap fails, the shadow table is dropped and the original is untouched. Interactive — click or press Enter ({{ opened }})
