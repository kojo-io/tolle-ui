Theming in Tolle UI

Tolle UI is built with theme-first principles. Instead of hard-coding colors into components, everything is driven by semantic tokens, CSS variables, and Tailwind utilities, giving you full control over branding and appearance.

Design Philosophy

Tolle UI uses foreground and background tokens instead of direct colors.

Example

  • bg-background
  • text-background

This abstraction ensures:

  • Consistent theming across components
  • Easy dark mode support
  • Seamless brand customization

You never theme components directly first — you theme the system, and components inherit automatically.

Tolle UI is not about chasing trends. It’s about making day-to-day Angular development calmer, clearer, and more predictable.

Global Theme Configuration

Tolle UI provides a theme provider that controls your application’s core design tokens.

Add this to your app.config.ts

>_ app.config.ts

Configuration Options

Option Description
primaryColor Base brand color used to generate all theme shades
radius Global border radius for components
darkByDefault Enables dark mode on initial load

Adding the Tolle UI Theme File

Tolle UI ships with a global theme stylesheet that must be included in your project.

Update your Angular styles array:

>_ angular.json

This file:

  • Defines CSS variables
  • Registers background/foreground tokens
  • Enables dark mode support
