Installation

Add it with the CLI (you own the source), or import the components if you use the package.

Examples

Basic

Compose the menu from an input, a list, and one or more groups. Arrow keys, Home, End and Enter are handled for you, and tolle-command-empty renders only when nothing matches the query. For the modal ⌘K palette use tolle-command-dialog; for a form control that writes back to a model, use tolle-combobox.

Keywords and disabled items

keywords adds synonyms that match without being displayed — type invoice or payment and the Billing row still matches. A disabled item stays visible but cannot be chosen and is skipped by the arrow keys. A group heading hides itself once every item in it is filtered out, so you never get a stranded header.

Manual filtering

Set [shouldFilter]="false" to keep every rendered item visible and do the matching yourself — the shape you want for server-side search. Listen to queryChange, narrow your own array, and render the rows with *ngFor. Since no item is ever filtered out by the menu, tolle-command-empty never fires here — render your own empty state instead.
