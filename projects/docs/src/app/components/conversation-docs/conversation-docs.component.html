Installation

Add it with the CLI (you own the source), or import the components if you use the package.

Examples

Basic

Messages go inside tolle-conversation-content, which lives in the scrolling viewport. The scroll button is projected into its own slot outside that viewport, so it stays fixed to the frame while the transcript moves under it.

Empty state

tolle-conversation-empty-state renders nothing once the transcript reports content, so you can leave it in the template permanently and just drive [empty] from your message list. Anything you project into it renders under the description — a suggested prompt, say.

Following a live transcript

The rule is: never move the reader against their intent. New messages pull the view along only while the reader is within threshold px of the bottom. The moment they scroll up to re-read something, following stops and the jump button appears; scrolling back down resumes it.

New messages arrive as projected DOM rather than as inputs, so the transcript watches its own subtree with a MutationObserver — it works whatever renders the messages.
