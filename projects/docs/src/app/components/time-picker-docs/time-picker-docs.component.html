Installation

Add it with the CLI (you own the source), or import the component if you use the package.

The value is a 24-hour string

This is the first thing to know: tolle-time-picker exchanges a 24-hour HH:mm string — or HH:mm:ss with [showSeconds]="true" — and it is deliberately timezone-free. A wall-clock time has no calendar day and no UTC offset, and a Date would invent both when all the user did was pick “2:30 PM”.

If you really do need a Date through the form control, set valueType="date" — the time is then written onto today’s date, or onto the date already held by the control. When you need a real date and a time together, reach for tolle-date-time-picker instead; for a date alone, use tolle-date-picker.

Examples

Basic with ngModel

The picker implements ControlValueAccessor, so [(ngModel)] is all it takes. Writing a malformed string into the control simply clears the selection rather than throwing.

12-hour face with seconds

[use12Hours]="true" adds an AM/PM column and renders the trigger as a clock face — but the value stays 24-hour, so nothing downstream has to care. [showSeconds]="true" adds the seconds column and lengthens the value to HH:mm:ss; [secondStep] thins that column out.

Bounds, steps, sizes and invalid

min and max are themselves 24-hour HH:mm strings. Entries that could not possibly produce a legal time are disabled rather than hidden, so the columns keep their shape. Four trigger heights are available, plus [invalid]="true" for the destructive border and focus ring.

Reactive forms

Use formControlName like any other control; the control holds the same HH:mm string. Calling control.disable() reaches the picker through setDisabledState(), greys out the trigger and closes the panel if it happens to be open.

Building your own trigger

tolle-time-columns is the bare scrollable column strip — the option generation, the min/max disabling and the roving keyboard model — with no trigger and no popover of its own. Both tolle-time-picker and tolle-date-time-picker embed it, and so can you when you want the columns inline, in your own dialog, or under a trigger of your own design. It is not a form control: drive it with [hour], [minute] and [second], and listen to (timeChange). Its bounds are given as seconds since midnight rather than strings.

Time helpers

The parsing and formatting the picker uses internally are exported as pure functions, so your own validators and display code can agree with it exactly. parseTimeString returns null for anything malformed or out of range, and to12Hour / from12Hour get midnight and noon right, which a naïve hour % 12 does not.

Keyboard

  • Enter, Space, ↑ or ↓ on the trigger opens the panel and focuses the first column.
  • ↑ / ↓ move the active entry within a column, skipping disabled entries; Home / End jump to its ends.
  • ← / → move between columns.
  • Enter or Space commits the active entry — the roving highlight is styled apart from the committed value, so arrowing around never looks like selecting.
  • Escape closes the panel and returns focus to the trigger.
