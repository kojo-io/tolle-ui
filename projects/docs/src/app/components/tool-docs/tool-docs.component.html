Installation

Add it with the CLI (you own the source), or import the components if you use the package.

Anatomy

Compose the call from a header and its two payload panels. The tool owns the shared open state and the lifecycle state, so the header renders its chip and the panels show and hide themselves without bindings of their own.

Examples

Basic

A completed call. Hand the panels any value: objects are pretty-printed as JSON, strings render as they are. Both panels only exist in the DOM while the call is expanded.

States

state colours the header chip and the tool's border: muted while pending, info while running (the chip icon spins), and destructive on failure. Set error on the output to style the payload itself as a failure.

Driving the lifecycle

Move state from pending through running to success as the call resolves, and expand the tool with [(open)] so the result is visible the moment it lands.
