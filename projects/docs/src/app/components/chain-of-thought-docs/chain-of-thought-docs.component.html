Installation

Add it with the CLI (you own the source), or import the components if you use the package.

Anatomy

A header that toggles the trace, content that only renders while expanded, and one step per thing the agent did. A step can project search-result chips for what it found.

Examples

Basic

A finished trace. The rail connects the step markers and stops at the last one, so the list never trails off the bottom.

Live progress

Drive status per step as the agent works: complete behind it, active for the step running now (emphasised label, info-coloured marker), and pending ahead of it.

Collapsed with search results

The trace is collapsed by default, which is what you want under a finished answer. Chips summarise what a search step turned up, and size="sm" keeps the whole thing quiet next to the message it belongs to.
