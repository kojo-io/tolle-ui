Installation

Add it with the CLI (you own the source), or import the component if you use the package.

The value is a Date

Unlike tolle-time-picker, which exchanges a timezone-free HH:mm string, this control's value is a real Date — because a date-time genuinely is a moment. It emits null when the value is cleared with the panel's Clear button.

The behaviour worth calling out: changing the date preserves the chosen time, and changing the time preserves the chosen date. The two halves are only ever recombined, never reset — which is what people assume and rarely get. Picking a day before any time has been chosen starts that day at midnight; picking a time before any day has been chosen lands it on today.

Reach for tolle-date-picker when only a date is needed, and tolle-time-picker when only a time is.

Examples

Basic with ngModel

The picker implements ControlValueAccessor, so [(ngModel)] is all it takes. Open it and change the day — the 09:30 you started with survives. Press Clear and (valueChange) reports null.

Reactive forms

Use formControlName like any other control; type the control as FormControl<Date | null>. Calling control.disable() reaches the picker through setDisabledState(), greys out the trigger and closes the panel if it happens to be open.

Bounds, sizes and invalid

[min] and [max] are Dates carrying both halves. Days outside the window are disabled in the calendar, and the time half of the bound only applies on the boundary day itself — on the min day the columns start at the min time, on any later day they start at midnight. displayFormat is a date-fns pattern for the trigger text.

Anatomy and keyboard

The panel is tolle-calendar on the left and the tolle-time-columns strip on the right — the same strip tolle-time-picker uses, so the option generation, bound-disabling and keyboard model are shared rather than written twice. If you want those columns under a trigger of your own, embed tolle-time-columns directly; it is documented on the time picker page. The footer holds Clear, which sets the value to null and closes, and Done, which closes and returns focus to the trigger.

  • Enter, Space, ↑ or ↓ on the trigger opens the panel.
  • Inside the time columns, ↑ / ↓ move within a column (skipping disabled entries), ← / → move between columns, and Enter commits.
  • Escape closes the panel and returns focus to the trigger.
