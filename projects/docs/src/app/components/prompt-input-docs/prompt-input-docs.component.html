Installation

Add it with the CLI (you own the source), or import the components if you use the package.

Anatomy

The composer owns the textarea; everything below it is yours to project. The toolbar splits into a left tool cluster and a right-aligned submit button, and the submit button reads the composer's state on its own — no bindings between them.

Examples

Basic

It is a ControlValueAccessor, so [(ngModel)] works directly. Enter submits, Shift+Enter inserts a newline, and the textarea grows with the message up to maxRows before it scrolls. The send button stays disabled until there is text to send.

Streaming and stopping

Set status="streaming" while the response is in flight: the border turns informational and the send button becomes a red stop button that emits stopped. One control covers both directions, and stopping stays available even with an empty composer. Submitting is refused while streaming, so a stray Enter cannot double-send.

Reactive forms and submitOnEnter

formControlName binds like any other control, and disabling the control disables the composer. Here [submitOnEnter]="false" makes Enter always insert a newline — a better fit for a long-form draft, where sending should be deliberate.

Keyboard

  • Enter — submits, while submitOnEnter is true and the composer is neither empty nor streaming.
  • Shift+Enter — inserts a newline. Alt, Ctrl and Cmd chords are left alone too, and an in-progress IME composition never has its Enter stolen.
