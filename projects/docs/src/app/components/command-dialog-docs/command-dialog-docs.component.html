Installation

Add it with the CLI (you own the source), or import the components if you use the package.

How it works

tolle-command-dialog is only a shell: a backdrop, a centred panel, and the key handling. The menu itself is projected — you put a full Command menu inside it, with its own input, list, groups and items. Everything you can build with tolle-command works here unchanged.

The dialog binds a global hotkey. By default shortcut="mod+k" — ⌘K on macOS, Ctrl+K everywhere else — toggles the palette from anywhere on the page. Escape always closes it, regardless of the shortcut, and shortcut="" disables the key binding entirely so you can drive the palette yourself through [(open)].

Because the binding is global, every mounted dialog listens on document — two dialogs sharing a shortcut will both react to it. That is why the second example below uses mod+j: both demos are live on this page at once, and reusing mod+k would open them together. In a real app, mount one palette at the root of your layout.

Examples

Default ⌘K palette

Press ⌘K (or Ctrl+K) anywhere on this page, or use the button. Choosing an item emits its value from the menu's selected output — close the palette yourself in that handler.

Custom shortcut, no backdrop close

This palette answers to ⌘J / Ctrl+J and ignores backdrop clicks, so it can only be dismissed with Escape or the close button in its footer. The status line reflects the opened and closed outputs, which fire when the dialog changes state itself — via the hotkey, Escape, or the backdrop — not when you set open from outside.
