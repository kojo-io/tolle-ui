Installation

Add it with the CLI (you own the source), or import the component if you use the package.

Behaviour

A menu bar behaves differently from a row of independent dropdowns. Once any menu is open, hovering a different trigger switches straight to it — no second click. Hovering while everything is closed does nothing.

Keyboard support comes for free: Arrow Left / Right move between menus, Arrow Up / Down move between the items of the open menu, and Escape closes the menu and returns focus to its trigger. Clicking outside the bar closes it too.

Examples

Basic

A classic File / Edit / View bar with a label, separators, and a destructive item. Each item reports itself through (select).

Sizes

Three scales. Every trigger inherits the bar's size unless it sets its own.

Icons and shortcuts

Items project a prefix slot before the label and a shortcut slot after it. Use inset to line up items that have no icon.
